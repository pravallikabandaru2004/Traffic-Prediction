Needs of the Project

Unpredictable Congestion

Urban traffic is non-linear and chaotic. Traditional statistical models fail to capture sudden spikes caused by accidents or weather.

Hardware Limitations

Traffic cameras (Edge Devices) have limited processing power. Complex Graph Neural Networks (GNNs) are too heavy to run directly on these devices.

Real-Time Latency

Traffic light optimization requires milliseconds of latency. Cloud-based processing is often too slow; we need a lightweight model for instant prediction.

Importance

🚀 30x Faster Inference

By replacing graph convolutions with pure MLP matrix operations, our ST-MLP model achieves 30x faster prediction speeds while maintaining accuracy.

🌍 Sustainability

Efficient traffic flow reduces vehicle idling time. This directly translates to lower fuel consumption and reduced CO2 emissions in smart cities.

🔌 Infrastructure Agnostic

Unlike GNNs, this model doesn't need a pre-defined graph of the road network during inference, making it incredibly easy to deploy in new cities.

Modules Implemented

1. Authentication Module

Handles secure user registration and login using SQLite and Flask sessions to protect the dashboard.

2. Spatial Interaction Module

Integrates Leaflet.js to allow users to click real-world coordinates on a map and automatically identify the nearest traffic sensor.

3. Data Fusion Module

Merges dynamic time-series flow data with static metadata (Latitude/Longitude) before feeding it into the AI model.

4. ST-MLP AI Module

The core engine. Utilizes TempEncoder for time-series encoding and ST-Mixer layers to capture spatial dependencies without graphs.

5. Visualization Module

Generates real-time plots comparing historical traffic data against the model's future predictions.
